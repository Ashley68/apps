Python Reference for web applications

Python Overview

  • About Python:
    • Python is a relatively new language, released in 1991. Guido van Rossum was the principal author, and he plays a central role in overseeing the directon of the language. Organizations that use Python as their primary or a primary backend programming language include Google, Youtube, NASA, Reddit, Quora, Dropbox, Yahoo, Instagram, Pinterest.
    • Python is a general purpose programming language, which includes web programming but is used in many other capacities, and is popular in scientific communities.
    • It is an object oriented programming languange meaning it is data-centered as opposed to function centered.
    • A script is a list of commands that can be executed without user interaction. Python is not specifically a scripting language like a pure web language such as PHP or JavaScript, but it's web programs are scripts.
    • Python is an interpreted language as opposed to a compiled language like C or C++. This means that the source files can be run directly without first creating an executable file. Because you don't have to take the step of compiling it, there are less steps involved in developing and testing. So in that sense, interpreted languages are easier for development. However, compiled languages optimize the code so they usually run faster.
    • Although Python is an interpreted language, Python converts it's files into byte code the first time you run them. Those are the files that end in pyc. This strips out unnecessary code like spaces and comments to make Python code run fast.
    • Python stands out among programming language for its explicit use of white space (spaces and blank lines). While most programming languages mostly ignore white space, and rely on things like colons, semicolons, curly braces etc. to distinguish between lines of code, Python generally uses indentation and blank lines to make these distinctions. As a result there are less arcane characters to type and mistype, and the layout of the code is very consistent, clean, and readable. This has contributed greatly to the language's popularity.
    • Python's documentation is considered very good. The website is www.python.org
      • , with the documentation at www.python.org/doc
  • Python versions:
    • At the time of this writing, the current version of Python was 3.3. The latest 2.x version was 2.7. The Django framework was not compatible with Pyton 3.x until the release of Django 1.5 in Feb 2013. Django 1.5 is compatible with both Python 3.3 and 2.7. So which should you use? More and more Python applications are converting to 3.x. But many third party site packages may not work properly with Python 3.x.
    • You can actually download both and run either on your machine to test code. But you have to take a couple of steps. Obviously the Python folder names will be different, like python27 and python33. You don't want to run both versions of Python at the same time so you need to change your Path from Python27 to Python33 or viceversa. Additionally, at least on Windows machines you should open the verion you are not using and change the .exe file names. So if you want to use python 2.7 and not python 3.3 then go to the Python33 folder and change python.exe to python33.exe, and pythonw.exe to pythonw33.exe (or could be any name really). Your system will search for Python.exe so by changing the name of one set it won't execute the wrong one.
  • Official Python glossary of terms and of symbols.
  • Python concepts you need to be familiar with: variables, strings, numbers, lists, tuples, dictionaries, functions, classes, modules and packages, loops, conditionals (e.g., if statements), and regular expressions.

General Syntax and Style

  • Python has an official style guide called Pep8 found at www.python.org/dev/peps/pep-0008. Some things are required to make your code work (such as the right number of spaces), and others are good practices, but your code will still work without following them. But in general... follow them.
  • Use 4 spaces for each indentation level. Don't use tabs for indentation.
  • Limit all lines to a maximum of 79 characters.
  • Comments:
    • Single line comments: start with the pound sign # enter single line comment here
    • For multiple line comments, you can either put a # in front of each line, or start and end the comment block with three quotation marks """ enter multiple line comments here """
    • Comments should describe why you are doing what you're doing. They should help another programmer, or you if you are reading your code months or years later, understand what the code is for. Keep them relatively short and to the point.
  • Each category below includes the syntax and style that applies to it.

Variables and Constants

  • Variable: A symbol or name that stands for (represents) a value. Example: first_name = "Steve". The variable's value can be changed at any time.
  • Structure:
    • The equal sign is used to assign values to variables.
    • The operand to the left of the = operator is the name of the variable. Variable names can include letters, numbers and underscores, but cannot start with a number. Use lower case letters. If more than one word is needed, separate them with underscores. Functions use this same naming convention.
    • The operand to the right of the = operator is the value stored in the variable.
    • Variables must have a data type. The data type does not need to be explicitly declared (variable x is a string). Rather it is implied by the syntax around the value. If the value is a string, put it in quotes. A number without quotes is an integer, with quotes it is a string, and with decimal points it is a float. Multiple values inside square brackets are a list. Multiple values in parentheses or with no kind of brackets at all are tuples. Values in curly braces are dictionaries. A single word or word with underscores, but without quotes implies that you are assigning another variable to this variable variable1 = variable2.
  • Scope:
    • The variable's scope is how much of your program it applies to. A global variable applies to your entire program. A local variable applies to just one part of your code. For instance it can apply only to one function.
    • Whether a variable is global or local depends on where you place the variable. If you put it inside a class or function then it applies only to that class or function. You can add some code that passes the variable on to another part of your code too. For instance the user's username can be passed from a function to a template, so that the username can be displayed (hello username).
  • Namespace: a namespace is where a variable is stored in Python's memory.
  • Constant: A constant is like a variable but once it is set it doesn't change. The same structure that applies to variables applies to constants, except that CONSTANTS should be in ALL CAPITAL LETTERS to distinguish them from variables.

Datatypes

  • Data type is a "category" of data that differ in their properties (such as mutability and immutability), the methods and functions applicable to them, and in their representations. Python has built-in data types for stings, numbers, tuples, lists and dictionaries.
    • Mutable objects can change their value but keep their id(), while immutable objects have fixed values that can't be altered.
  • String data type:
    • A string is a series of characters in single or double quotes, often used to display text to the user. Example: error_message = "You must enter a username". In this example error_message is a variable name that is set equal to a string.
    • Python lets you put variables into strings by using the percent symbol as a placeholder. To put a string variable in a string use %s. Example: "My name is %s." % "Steve". Or if the variable is defined elsewhere, put the variable name without quotes. Example: "My name is %s." % first_name
    • To put a digit variable in a string: use %d. Example: "There are %d items on the list." % 10 or if the number if items is defined elsewhere in a variable called item_quantity: "There are %d items on the list." % item_quantity.
    • If you have multiple variables in one string, put them in parentheses, separated by commas. Example: "There are %s articles published and %d articles unpublished." % ('five', 2)
    • To display a string on multiple lines you can use \n. Example: weekdays = "Mon\nTues\nWed\nThurs\nFri".
    • To display a string on multiple lines you can also put it inside triple quotes. Example: 
      Days = """
Mon
Tues
Wed
"""
    • To display a string starting with a tab use \t. Example "\tMy name is Steve."
    • To escape special characters like quote use the forward slash \. Example: quote_variable = "The politician said \"Don't quote me on that\"."
    • You could also put it in triple quotes: quote_variable = """The politician said "Don't quote me on that"."""
  • Common Numeric data types:
    • Integer: a whole number. Has no decimal points and can be positive or negative.
    • Float (or Floating Decimal Point): A number with decimal points.
    • Boolean: data type that can only be True or False. Boolean is actually a subtype of Integer.
  • List:
    • A list is a sequence of values (i.e., a list of values). It can contain strings or numbers. The values of a list are mutable, meaning they can change. Other programming languages call this an array.
    • Lists have square brackets. List items are separated by commas. If list items are strings put them in quotes.
    • Format: list_name = [value1, value2, value3]
    • Example: students = ['Mike', 'Lisa', 'Victoria', 'Tom']. Since there are new students every semester, the values of the list change.
  • Tuple:
    • A tuple is a list that you can't modify (it's immutable). Once you set it, the values won't change.
    • Instead of using square brackets like a list, tuples use parentheses. If a list has no brackets, it will default to a tuple. Separate values in a tuple with commas. If there is only one value in a tuple, you put a comma after it.
    • Format: tuple_name = (value1, value2, value3)
    • Example: weekdays = ('Mon', 'Tues', 'Wed', 'Thurs', 'Fri')
  • Dictionary:
    • A dictionary (also called a dict) is a list of key and value pairs. The main operations on a dictionary are storing a value with some key and extracting the value given the key. Other programming languages call these associative arrays.
    • The keys must be unique to the dictionary, but the values don't have to be. The key must be of one data type (strings, numbers, or tuples) and immutable (you can't change the keys). The values don't have to be unique and they are mutable (they can change).
    • Dictionaries are surrounded by curly braces. Each key and value is separated by a colon, and each pair is separated by a comma.
    • Format: dict_name = {key1:value, key2:value, key3:value}
    • Example: gadget_prices = {'laptop computer':2000, 'desktop computer':1000, 'printer':500, 'smart phone':300}

Modules and Packages

  • Module: A module is a file that contains variables, functions and classes related to a specific function or activity. For instance there are Authentication modules, Time and Date modules, Math modules, etc. Other programs call these libraries, and in Python they are also sometimes called libraries.
  • A Package is complilation of related modules. A package must include a file named __init__.py. The presence of this file tells Python that this is a Package. The file may or may not contain any code it it, but other files in the directory will contain functions and classes that make up the package. Apps are a type of package.
  • Module and Package sources:
    • A) Built-in Python modules and packages:
      • There are modules and packages built-in to the Python Standard Library that is installed with Python. Some examples include datetime, math, os.path for common path manipulations, string for string manipulations, sqlite3 for interfacing with that database, email, json, and more. The full list in on the Python website library docs page.
      • Some modules are written in Python and some in C. The ones in C you can't view. The Python ones are located in the Python/lib folder.
      • For the full list of built-in functions go to the Python website functions page.
      • You can view a list of all the build-in modules and packages through the command line. Enter python -v and the list will print on your command line interface.
      • Within a particular module you can view all it's functions and classes through the command line as well using the dir() function. Using the Math module as an example, go to the command line and enter python --> import math --> print dir(math)
    • B) Framework Modules and Packages:
      • Frameworks like Django are installed in Python's Site-Packages folder. Here's the path: Python/Lib/Site-Packages/.
      • But frameworks will have a number of modules within the framework. So Django modules are located within the Django folder Python/Lib/Site-Packages/django/.
      • Django has built-in modules related to user authentication, Caching, Logging, Sending emails, Syndication feeds (RSS/Atom), Comments, Pagination, Messaging, Sessions, Static files management, Data validation and more. To see more go to docs.djangoproject.com.
    • C) Third Party Packages:
      • Third parties create packages for Python such as pip, virtualenv, and virtualenv wrapper. There are also third party packages that plug into web frameworks like Django such as Django-Registration and Django-Contact to make registion and contact forms that work with Django.
      • There is a comprehensive list of all Pyton Packages including Django Apps indexed on Python's website https://pypi.python.org. There is also a site just for Django packages https://www.djangopackages.com/, but not all the Django packages are there.
      • Packages are installed in the site-packages folder located at Python/Lib/Site-Packages/.
      • You can use pip or easy-install from the command line interface to quickly install third-party packages.
    • D) Modules that you create yourself.
      • When you create a function or class and save it to a file so it can be imported and used in another file, you have in essence created your own module.
  • Importing modules:
    • You need to import the function or class from its module or package when you call it from a file.
    • Import Structure:
      • The format is from module_name import function_or_class_name.
      • All imports go at the top of the file. List each import on a separate line. If there are multiple functions imported from the same module you can include them on the same line, separated by commas from module import function1, function2
      • Don't put the .py in the module names. If the module is forms.py you just put forms.
      • For Django modules include django in front of the module name using dot notation. Example: from django.db import models
    • List the imports in the following order based on the source of the module. Examples are included:
      1. Standard Python Library imports. Example: from math import sqrt
      2. Core Django (or other framework) module imports. Example: from django.shortcuts import render_to_response
      3. Third Party app imports.
      4. Imports from apps or files that you created in the project. Example: from myapp.forms import UserProfileForm
    • Python will search for the modules in the current folder, in Site-Packages folder, and in the PYTHONPATH. PYTHONPATH is not created by default. You create it and store it with your other Enviromental Variables like PATH only if you are storing modules outside where Python would orginarily look.
    • If the module is in the same folder as the file you are importing it to, then you leave out from and just include the import import module_name.function_or_class_name

Functions

  • A function is a mini-script that executes one or more related commands.
  • You first define the function. That means write the code that makes it work. Then you can call the function from somewhere else in your code. Could be from another function. Calling the function means running it, and passing it any input (aka arguments) it needs to return an output.
  • Function sources:
    • There are functions built-in to the Python language like open() which opens a file. For the full list of built-in functions go to the Python website functions page.
    • There are functions that are part of a framework like Django, or from other site packages. You need to import these before using them.
    • Then there are functions that you write yourself called user-defined functions. These can be in the same file they are called from, or you can put them in separate files, then import them before calling them.
  • If there is more than one function in a file, separate the functions with two blank lines.
  • Function Structure: 
    # Imports, if any, go at the top of the file, followed by two blank lines.
from source import module_or_function_name


def function_name(parameter1, parameter2):
    # doc statement explaining what the function does.
    statements
    return statement
  • Imports:
    • Any imports used in a function should be listed at the top of the page.
    • List imports on separate lines. Put them in the following order:
      1. Standard Python Library imports.
      2. Core Django module imports.
      3. Third Party app imports.
      4. Imports from apps or files that you created in the project.
  • The function itself is a block of code.
  • First line:
    • The first line of a function includes def (for define), the function name, in parentheses any parameters that you give it, then a colon
    • A function is roughly analogous to a verb, so in general, use a verb as the function name. Best is a command type of verb like save() or close(). The function name should be lowercase. Keep the name short, but if multiple words are needed, separate them with underscores. The name can have numbers in it but cannot start with a number.
    • Parameters are like variable names that are placed in parentheses after the function name. Arguments are the values that are plugged into the parameters when you call the function. The terms parameter and argument are often used interchangeably, even though they are not exactly the same thing.
    • If there are multiple parameters/arguments, separate them by commas.
    • If there is an undefined number of arguments you can put an asterisk before the parameter name.
    • If there are no arguments you just put empty parentheses.
  • Optional Documentation String:
    • Below the opening line you can insert a documenation string as a comment. This documents what the function does. It's not required but useful, especially if other programmers will be reading your code.
  • Function statements:
    • A statement is an instruction, written in a line or block of code.
    • All the statements that are part of the function must be indented four spaces.
    • Some (not all) functions include return statements that return a result as the output of the function. Once a result is returned, the function terminates.
  • Calling a function:
    • You call a function by using it's name followed by parentheses. If there are any arguments (i.e., the values you plug into the function), you put them in the parentheses separated by commas function_name('argument1', 'argument2')
    • Call, run, use, and execute a function all mean the same thing.
    • If the program calling the function is in a different file than the function, you need to import it. Put imports at the top of the file: from foldername.filename import function_or_class_name. For the filename you leave out the .py. function_or_class_name is the name of the function or class that's being imported.
    • When you call an imported function you need to use dot notation filename.function_name(). With arguments: filename.function_name('argument1', 'argument2')
  • Decorator: A decorator is a function that modifies another function or method. Python decorators begin with @ and are put in the line directly above the function they are modifying. As an example, the Django framework has a handy decorator that restricts pages to logged in users only. In the file that contains the function to open a page, you first import the decorator function from django.contrib.auth.decorators import login_required. Then insert the decorator above the function. 
    @login_required 
def open_restricted_page(request):
    return render_to_response('restricted_page.html')

Classes

  • A class is a grouping of functions and variables that are closely related. They are grouped under a class so they can interact with each other.
  • Python is an object oriented programming languange. An instance of a class is an object, and an object contains values (defined in the class attributes) and has a defined behavior (defined in the class methods).
  • If there is more than one class or function in a file, separate them by two blank lines.
  • Class Structure: 
    # Imports, if any, go at the top of the file, followed by two blank lines.
from source import module_or_class_name


class ClassName(object):
    # Doc statement describing the class.
    attributes

    methods
  • First line of a class:
    • The first line of a class includes class, the class name, in parentheses object or the name of a parent class you are inheriting from, then a colon
    • If a function is loosely analogous to a verb, then a class is loosely analogous to a noun. The class name generally will be a noun. Class names should be in CamelCase. That is first letter of each word is capitalized, and if there are more than one word they are put together with no space between them.
    • You can add on to an existing class by inheriting it. You create a new class called a child class or subclass, and put the name of the parent class inside the parentheses. In the new class will use all the code from the parent class, and will add to it or modify it with the code in the new child class.
    • If you are not inheriting a class, you put the word (object) in the parentheses. A class without any inheritance is called a base class.
  • Body of the class:
    • Functions that are inside a class are called methods. Separate methods in a class with one blank line.
    • A base class will often have it's first method name as def __init__(self, argument2, argument3):. __init__ tells Python to initiate an instance of the class.
    • The argument self is used in Python to mean the current instance of this class.
    • Variables inside a class are called attributes.
  • Calling a class:
    • You call a class the same way you call a function. Use it's name, followed by parentheses. You can tell whether the call is to a function or a class because a function_name will be all lower case, with underscores between words, while a ClassName will be CamelCase.
    • If the class is defined in a separate file you have to import the class at the top of the file.

Conditionals (e.g., If Statements)

  • An if-statement creates a branch in the code. If the boolean expression is True then run the code under it, otherwise, skip over it.
  • The if statement starts with if condition:
  • This is essentially saying if the condition is true, or if it exists, then run the the code below it. While you don't explicitly compare the conditional to true if condition == True:, that is sort of or in some cases exactly what you are doing.
  • The if statement clauses are placed below the if condition: line, and must be indented four spaces.
  • If statements should be followed by an else: statement that runs if the if statement is false.
  • You can include multiple if conditionals by including
  • elif
    • statements. Python will go down the list in order until one is true, then it will run the code.
  • Nested if statements are if statements inside of if statements.
  • if statement structure: 
    if condition:
    do this
    and this
elif condition:
    do this
else:
    do this if the above conditions are false.

Loops

  • A loop runs a block of code over and again. A for loop runs a block of code for each item in a list.
    • For loop: If you want to do something with every item in the list, you can use a for loop. Syntax: A variable name follows the for keyword; it will be assigned the value of each list item in turn. in list_name designates list_name as the list the loop will work on. The line ends with a colon (:) and the indented code that follows it will be executed once per item in the list. for loops allow us to iterate through all of the elements in a list from the left-most (or zeroth element) to the right-most element.
  • For loop format: 
    for variable_name in list_name:
    do this
  • You can create the variable_name just for this for loop. The variable_name will be assigned a new value every time the loop goes through the list.
  • For example, you can use a for loop to display all the records from a database.
  • You can also limit the output by including an if statement in the for loop. 
    for variable_name in list_name:
    if condition:     
        do this
  • You can create a new list, then populate it with a for loop. To do that start by creating a variable name and assigning it to an empty list listname = []. Then run the for loop, and assign the results to the list name.
  • A while loop is similar to a for loop, but it does something while a condition is true. It runs the danger of creating an endless loop. For that reason it's better to use a for loop if you can, but there are occasions when a while loop may be the best option.
  • While loop format: 
    while condition:
    do this

Python Operators

    An operator is a symbol that represents a specific action, such as a plus sign representing addition.
Arithmatic Operators:
+ addition
- subtraction

/ division
% modus. Divides left hand operand by right hand operand and returns remainder. Examples: 52%10 returns 2; 16%10 returns 6
** exponent

Comparison Operators. Also called Boolean Operators because they return True or False. Used in conditional statements:
< less than
<= less than or equal to
> greater than
>= greater than or equal to
!= not equal to
== equal to. Use two equal signs. One equal sign is used to assign values to variables.
Assignment Operators:
    Below is a simple assignment operator, and four contraction operators.
= Assigns values from right side operands to left side operandvariable_name = value
+= Contraction of Addition and Assignment Operators. Adds right operand to the left operand and assigns the result to left operandExample: a+=b is a shortcut way to add b+a then assign the result back to a.
-= Contraction of Subtraction and Assignment Operators. Subtracts right operand from the left operand and assigns the result to left operandExample: a-=b is a shortcut way to subtract b-a then assign the result back to a.
*= Contraction of Multiplication and Assignment Operators. Multiplies right operand by the left operand and assigns the result to left operandExample: a*=b is a shortcut way to multiply b*a then assign the result back to a.
/= Contraction of Division and Assignment Operators. Divides left operand with the right operand and assigns the result to left operandExample: a/=b is a shortcut way to divide c/a then assign the result back to a.

Regular Expressions (Regex)

  • A regular expression (sometimes abbreviated as regex) is a formula for matching strings based on a pattern. It can be used to test input to a website to make sure it matches the input criteria. For instance it can check a name to make sure it only contains letters and spaces. It can check a password to ensure it contains numbers, letters and some special characters. Django uses it to check URLs for a specific pattern. If it matches it will route the page a certain way, if not it checks the next url regex pattern.
  • Python has a regex module called re that is part of it's standard library. To use it you insert an import at the top of your file import re. Django uses this module as part of it's urls module. Django's urls.py file starts with from django.conf.urls import patterns, include, url. This import has Python re module is embedded in it.
  • A full list of Python's regex syntax is at http://docs.python.org/3.3/library/re.html
